1. Intake

LLM (Sonnet 4.6)

Uppgift

Extraherar projektinfo från fri text. Identifierar byggnadstyp, area (BTA) och renoveringskomponenter.

Nyckelregler

  • Frågar aldrig om materialval
  • Får fråga om: byggnadsår, krav, total/partiell, budget
  • Max 1-2 förtydligande-frågor
  • Skattar mängd om ej angiven

Input

Fri text på svenska, t.ex.: "Vi ska renovera en skola på 500 kvm. Nya golv, byta fönster och ny ventilation."

Output

Project JSON med building_type, area_bta, components[], clarification_needed

Kategorier: golv, vägg, tak, fönster, dörr, installation, isolering, övrigt

2. Baslinje

Boverket + LLM-fallback Websök (pris)

Uppgift

Beräknar klimatpåverkan och kostnad för standardmaterial per komponent. NollCO2-inspirerad baslinje med Boverkets Typical-värden (A1-A3).

Datakällor

  • Klimat: Boverkets klimatdatabas (Typical A1-A3)
  • Pris: Webbsökning via Sonnet 4.6 + thinking (OpenRouter)
  • Fallback: Lokal climate_data.py

Input

Project med komponentlista

Output

Baseline med per komponent: co2e_kg, cost_sek, source (klimat), cost_source (pris)

Separata källkolumner för klimat och pris.

3. Alternativ

LLM + EPD-data Websök (pris)

Uppgift

Tar fram klimatsmartare alternativ per komponent. LLM:en resonerar som expert kring förkategoriserade EPD:er. Återbruksalternativ kompletteras från lokal data.

Nyckelregler

  • Förkategoriserade EPD:er per komponentkategori i prompten
  • LLM:en väljer 2-4 mest relevanta per komponent
  • Prissökning parallelliserad (ThreadPoolExecutor, 5 trådar)
  • Svenska/nordiska produkter prioriteras

Input

Project + Baseline + förkategoriserade EPD:er (222 st, 13 kategorier)

Output

AlternativesResult med ComponentAlternatives[] + commentary

Varje alternativ: namn, CO2e, kostnad, källa, motivering, typ (återbruk/klimatopt.)

4. Sammanställning

Ren beräkning

Uppgift

Aggregerar användarens val till totaler. Jämför valt scenario mot baslinjen.

Beräkningar

  • Total CO2e och kostnad (valt scenario)
  • Total CO2e och kostnad (baslinje)
  • Besparing i kg CO2e och procent
  • Kostnadsskillnad i SEK

Input

Project + Selections (användarens val per komponent)

Output

AggregateResult med totaler, besparingar och per-komponent-jämförelse

5. Rapport

LLM (Sonnet 4.6)

Uppgift

Genererar beslutsunderlag i markdown. Sammanfattar projektets klimatpåverkan, valda alternativ och potentiella besparingar.

Format

  • Sammanfattning med nyckeltal
  • Per-komponent-tabell med baslinje vs valt
  • EPD-referenser och källhänvisningar
  • Rekommendationer och nästa steg

Input

Project + Selections + all aggregerad data

Output

Markdown-rapport renderad i resultatpanelen

Dataflöde

Prisdata
Webbsökning via LLM

Modell

Claude Sonnet 4.6 via OpenRouter med 5000 tokens thinking budget (medium). Webbsökning aktiverad (max 3 anrop per produkt).

Process

Söker aktuella priser hos svenska bygghandlare (Byggmax, Beijer, XL-Bygg). Returnerar marknadspris i SEK exkl. moms. Parallelliserat med ThreadPoolExecutor (5 trådar).

Cache

price_enriched-flagga i cache förhindrar omhämtning av redan prissatta poster. Priser lagras tillsammans med klimatdata i samma cache.

Fallback

Om OpenRouter-nyckel saknas eller webbsök misslyckas returneras cost_sek: 0 och cost_source: "". Prisdata är kompletterande, inte blockerande.

Teknik
Stack

Backend

Flask (Python). REST API med 5 endpoints. Supabase JWT-auth (ES256/JWKS).

Frontend

Inline HTML/CSS/JS i Flask-template. Split-layout: 40% chatt, 60% resultat. Responsiv design med mobilvänligt stegflöde.

LLM

Claude Sonnet 4.6 via Anthropic API (intake, alternativ, rapport). Sonnet 4.6 via OpenRouter med thinking + websök (prisdata).

Klimatdata

Boverkets API (baslinje) + Environdec EPD:er (alternativ) + lokal fallback. Layered provider: Boverket → Environdec → lokal.

Deploy

Vercel med GitHub auto-deploy. Repo: henricbarkman/aida-klimatkalkyl. Syncat från generalassistant.

Auth

Supabase (primärt) med JWKS auto-discovery (ES256). Lösenord som legacy-fallback.
