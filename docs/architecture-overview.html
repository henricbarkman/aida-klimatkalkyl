1. Intake-agenten

LLM (Sonnet 4.6)

Uppgift

Extraherar projektinformation från fri textbeskrivning på svenska. Identifierar byggnadstyp, area (BTA) och renoveringskomponenter.

Nyckelregler

  • Frågar INTE om materialval. Fokus på behov, inte material.
  • Får fråga om: byggnadsår, särskilda krav (Miljöbyggnad, ljud, fukt), om renoveringen är total/partiell, budget/tidplan.
  • Skattar quantity om den inte anges.
  • Max 1–2 förtydligande-frågor, bara när det väsentligt förbättrar analysen.

Input

Fri text från användaren, t.ex.:
“Vi ska renovera en skola på 500 kvm. Nya golv i hela byggnaden, byta fönster och ny ventilation.”

Output

JSON med building_type, area_bta, components[], clarification_needed.

Kategorier: golv, vägg, tak, fönster, dörr, installation, isolering, övrigt.

Systemprompt (sammanfattning)Du är AIda, en assistent för klimatkalkylering av ombyggnationer i Karlstads kommun. Identifiera: byggnadstyp, area BTA, renoveringskomponenter. Fråga INTE om materialval — materialval är alternativ-stegets uppgift. AIda kan komma med bättre förslag än användaren tänkt sig. Fråga däremot gärna om byggnadsår, särskilda krav, total/partiell renovering, budget/tidplan. Svara med JSON. Komponent-ID: c1, c2... Enheter: m2, st, lm.

2. Baslinje-agenten

Lokal data + LLM-fallback

Uppgift

Beräknar klimatpåverkan enligt NollCO2-metodens baslinjeprincip: alla material köps nytt utan klimathänsyn. Ger det “värsta fallet” som referenspunkt.

Datahierarki

  • Först: Lokal databas (climate_data.py) — fördefinierade värden per komponenttyp
  • Sedan (LLM-fallback):
  • 1. EPD:er (environdec.com, EPD Norge)
  • 2. Boverkets klimatdatabas
  • 3. Uppskattning (markeras tydligt)

Input

Project-objekt med komponenter, byggnadstyp och area.

Output

Baseline med per komponent: co2e_kg, cost_sek, method, source.

Källmärkning: [Verifierad] för lokal data, [Uppskattning] för LLM.

Systemprompt (sammanfattning)NollCO2-metoden: Baslinjen = alla material köps nytt utan klimathänsyn (worst-case). För varje komponent: välj det alternativ med HÖGST CO2e. Datakällor i ordning: EPD:er → Boverkets klimatdatabas → Egen uppskattning. Svara med JSON. Ange alltid källa. Var ärlig om osäkerhet.

3. Alternativ-agenten

Lokal data + LLM + Kommentar

Uppgift

Hittar klimatsmartare alternativ för varje komponent. Minst 3 alternativ per komponent. Genererar också en sammanhängande kommentar om förslagen som visas i chatten.

Alternativtyper

  • Återbruk — Sola byggåterbruk, CCBuild m.fl. Om ej tillgängligt, sägs det explicit.
  • Klimatoptimerat — Minst 2 nyproducerade material med lägre CO2e. Konkreta produktnamn.

Kommentar (ny)

Efter att alternativ hittats görs ett extra LLM-anrop som skriver en kort (3–6 meningar) kommentar:

  • Lyfter intressanta alternativ
  • Nämner återbruksmöjligheter
  • Pekar på avvägningar (CO2 vs kostnad)
  • Helhetsintryck av besparingspotentialen

Output

AlternativesResult med per komponent: alternativ-lista + commentary (sträng).

Stödjer user_feedback för att begära fler alternativ.

Systemprompt (sammanfattning)Föreslå minst 3 alternativ med lägre klimatpåverkan per komponent: 1. Återbruk (om möjligt) 2. Klimatoptimerat nyköp — minst 2 olika Datakällor: EPD:er → Boverket → Uppskattning. Om inga återbruksalternativ finns, säg det explicit. Kommentarsprompt: Skriv som kunnig rådgivare till projektledare. Konkret, inte generiskt. Referera till faktiska material och siffror.

4. Sammanställningsagenten

Ren Python (ingen LLM)

Uppgift

Summerar användarens val: total CO2e, total kostnad, jämförelse mot baslinje, besparing per komponent. Rent deterministisk — inget LLM-anrop.

Beräkningar

  • Total CO2e + kostnad för valda alternativ
  • Total CO2e + kostnad för baslinjen
  • CO2e-besparing (kg + %)
  • Kostnadsskillnad (SEK)
  • Per komponent: valt alternativ, besparing, källa

Input

Project + Selections (användarens val per komponent).

Output

AggregateResult med totaler och komponentdetaljer. Varnar om någon komponent saknar val.

5. Rapportagenten

LLM (Sonnet 4.6)

Uppgift

Genererar strukturerat beslutsunderlag i markdown, lämpligt för tjänsteskrivelser och upphandlingsunderlag.

Rapportinnehåll

  • Sammanfattning överst
  • Komponenttabell med alla siffror + källor
  • Jämförelse baslinje vs valt
  • Osäkerheter och begränsningar
  • Disclaimer: uppskattningar för beslutsstöd

Stilregler

  • Formell svenska
  • Inga anglicismer
  • Inga AI-typiska fraser (“delve into”, “game-changer”)
  • Inga em-dashes
  • Siffror med enheter (kg CO2e, SEK)

Format

Markdown (primärt). PDF via Pandoc/XeLaTeX om tillgängligt, annars markdown-fallback.

Teknik

Backend

Flask (Python). REST API med 5 endpoints. Supabase JWT-auth (ES256/JWKS).

Frontend

Inline HTML/CSS/JS i Flask-template. Split-layout: 40% chatt, 60% resultat. Inga externa JS-beroenden.

LLM

Claude Sonnet 4.6 via Anthropic API. 4 av 5 agenter använder LLM. Aggregate är ren Python.

Klimatdata

climate_data.py — lokalt uppslagsverk för baslinjer + alternativ. LLM-fallback för okända komponenter.

Deploy

Vercel (planerat). Railway/Render som alternativ pga LLM timeout-gränser.

Auth

Supabase (primärt) med JWKS auto-discovery. Lösenord (legacy-fallback).
